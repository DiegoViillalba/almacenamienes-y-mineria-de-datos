El Problema Central

Alta confianza ≠ Relación fuerte

Una regla puede tener confianza = 0.90 y aun así no decirnos nada nuevo.

¿Por qué? Porque si el consecuente ocurre muy frecuentemente de todas formas…

\[\text{lift} = \frac{P(A \cup B)}{P(A) \cdot P(B)} = \frac{\text{confidence}}{P(B)}\]

Si \(\text{lift} \approx 1\): A y B son independientes. La regla no aporta (Tan, Steinbach, and Kumar 2006).

¿Qué Mide Realmente Cada Métrica?

Métrica Pregunta que responde Limitación
Support ¿Qué tan común es este patrón? No dice si es interesante
Confidence Si A, ¿qué tan probable B? Ignora la frecuencia base de B
Lift ¿Es la relación real o casual? Sensible a samples pequeños

Important

Regla de oro: Una regla útil necesita soporte razonable Y lift > 1 (Han, Kamber, and Pei 2011).

Cuándo NO Usar Reglas de Asociación

❌ Dataset pequeño → el soporte es inestable estadísticamente

❌ Ítems muy frecuentes dominan → distorsión por “whole milk effect”

❌ Buscas causalidad → las reglas son correlacionales, no causales (Pearl 2009)

❌ Necesitas interpretabilidad temporal → Apriori no captura secuencias

Tip

Alternativas: Para secuencias → GSP, PrefixSpan.
Para causalidad → modelos causales.

El Problema del Lift Máximo

El lift tiene un techo teórico que depende del soporte de los ítems:

\[\text{lift}_{\max}(A, B) = \frac{1}{\max(P(A), P(B))}\]

Code 
freq_rel = df.mean()  # soporte individual de cada ítem

# Calcular lift máximo teórico para cada regla
def lift_max(row):
    pa = freq_rel.get(row["antecedents_str"], np.nan)
    pb = freq_rel.get(row["consequents_str"], np.nan)
    if pd.isna(pa) or pd.isna(pb):
        return np.nan
    return 1 / max(pa, pb)

# Solo reglas con antecedente/consecuente simples (1 ítem)
reglas_simples = reglas[
    reglas["antecedents"].apply(len) == 1
].copy()
reglas_simples["lift_max_teorico"] = reglas_simples.apply(lift_max, axis=1)
reglas_simples["lift_relativo"] = (
    reglas_simples["lift"] / reglas_simples["lift_max_teorico"]
)

Bloque B: Métricas Complementarias

Más Allá del Lift: Tres Métricas Clave

La literatura propone docenas de métricas (Tan, Steinbach, and Kumar 2006). Estas tres son las más usadas en práctica:

Leverage (Apalancamiento) \[\text{lev}(A \to B) = P(A \cup B) - P(A) \cdot P(B)\]

Diferencia absoluta entre co-ocurrencia observada e independencia.
Rango: \([-1, 1]\). Leverage = 0 ↔︎ independencia.

Conviction \[\text{conv}(A \to B) = \frac{1 - P(B)}{1 - \text{confidence}(A \to B)}\]

¿Qué tan a menudo A ocurre sin B, relativo a lo esperado bajo independencia?
Rango: \((0, \infty)\). Conviction → ∞ cuando confidence → 1.

Zhang’s Metric \[Z(A, B) = \frac{P(A \cup B) - P(A)P(B)}{\max\{P(A \cup B)(1-P(A)), P(A)P(B \setminus A)\}}\]

Rango: \([-1, 1]\). Simétrico, robusto a ítems muy frecuentes.

Calculando las Métricas Complementarias

Code 
# mlxtend calcula leverage y conviction directamente
itemsets2 = apriori(df, min_support=0.10, use_colnames=True)
reglas_ext = association_rules(
    itemsets2,
    metric="lift",
    min_threshold=0.5
)

# Zhang's metric (manual — no está en mlxtend)
def zhang(row):
    pab = row["support"]
    pa  = row["antecedent support"]
    pb  = row["consequent support"]
    num = pab - pa * pb
    den = max(pab * (1 - pa), pa * (pb - pab))
    return num / den if den != 0 else 0

reglas_ext["zhang"] = reglas_ext.apply(zhang, axis=1)
reglas_ext["antecedents_str"] = reglas_ext["antecedents"].apply(
    lambda x: ", ".join(sorted(x))
)
reglas_ext["consequents_str"] = reglas_ext["consequents"].apply(
    lambda x: ", ".join(sorted(x))
)
reglas_ext["regla"] = (reglas_ext["antecedents_str"]
                       + " → " + reglas_ext["consequents_str"])

cols = ["regla", "support", "confidence", "lift", "leverage", "conviction", "zhang"]

Bloque C: Sensibilidad a Parámetros

Cuando ejecutamos:

Estamos tomando decisiones que no tienen justificación estadística intrínseca.

nodo_idx = {nd: i for i, nd in enumerate(nodos)}

# Layout circular
angulos = [2 * math.pi * i / n for i in range(n)]
xs = [math.cos(a) for a in angulos]
ys = [math.sin(a) for a in angulos]

# Aristas
edge_x, edge_y, edge_text = [], [], []
for _, row in reglas_grafo.iterrows():
    x0, y0 = xs[nodo_idx[row["antecedents_str"]]], ys[nodo_idx[row["antecedents_str"]]]
    x1, y1 = xs[nodo_idx[row["consequents_str"]]], ys[nodo_idx[row["consequents_str"]]]
    edge_x += [x0, x1, None]
    edge_y += [y0, y1, None]

# Importancia de nodo = suma de lifts de sus reglas
importancia = {}
for nd in nodos:
    r_sub = reglas_grafo[
        (reglas_grafo["antecedents_str"] == nd) |
        (reglas_grafo["consequents_str"] == nd)
    ]
    importancia[nd] = r_sub["lift"].sum()

fig = go.Figure()

fig.add_trace(go.Scatter(
    x=edge_x, y=edge_y,
    mode="lines",
    line=dict(width=1, color="#aaaaaa"),
    hoverinfo="none",
    showlegend=False,
))

fig.add_trace(go.Scatter(
    x=xs, y=ys,
    mode="markers+text",
    text=nodos,
    textposition="top center",
    marker=dict(
        size=[max(10, importancia.get(nd, 1) * 4) for nd in nodos],
        color=[importancia.get(nd, 0) for nd in nodos],
        colorscale="Plasma",
        showscale=True,
        colorbar=dict(title="Importancia<br>(suma lifts)"),
    ),
    hovertext=[
        f"{nd}<br>Importancia: {importancia.get(nd,0):.2f}"
        for nd in nodos
    ],
    hoverinfo="text",
    showlegend=False,

Leyendo el Grafo

Nodos grandes y brillantes: ítems que participan en muchas relaciones fuertes.

Nodos pequeños y oscuros: ítems periféricos — aparecen en pocas reglas o con lift bajo.

Ítems centrales vs. ítems puente:

  • Central (ej. whole milk): muchas conexiones, pero ¿son todas genuinas?
  • Puente (ej. granola): pocas conexiones, pero cada una puede ser muy informativa.

Warning

Un ítem muy central en el grafo no es necesariamente el más interesante.
Puede serlo simplemente por ser el más frecuente.

Generalización: La Regla del Consecuente Frecuente

Para cualquier ítem B con soporte \(P(B)\), el umbral mínimo de confianza para que una regla sea no-trivial es:

\[\text{confidence}_{\min} > P(B)\]

Y para que sea “sustancialmente” no-trivial con factor \(k\):

\[\text{confidence}_{\min} > k \cdot P(B)\]

Code 
umbrales = []
for item in df.columns:

        "item": item,
        "P(item)": round(p, 3),

Síntesis: El Pipeline Completo de Evaluación

Code 
def evaluar_regla(row, n_transacciones=50, verbose=True):
    """Evaluación sistemática de una regla de asociación."""
    veredicto = []
    puntos = 0

    # Criterio 1: lift
    if row["lift"] > 1.5:
        veredicto.append("✓ Lift fuerte (>1.5)")
        puntos += 3
    elif row["lift"] > 1.1:
        veredicto.append("~ Lift moderado (1.1–1.5)")
        puntos += 1
    else:
        veredicto.append("✗ Lift débil (≤1.1)")

    # Criterio 2: soporte absoluto
    n_obs = int(row["support"] * n_transacciones)
    if n_obs >= 10:
        veredicto.append(f"✓ Soporte sólido ({n_obs} transacciones)")
        puntos += 2
    elif n_obs >= 5:
        veredicto.append(f"~ Soporte mínimo ({n_obs} transacciones)")
        puntos += 1
    else:
        veredicto.append(f"✗ Soporte insuficiente ({n_obs} transacciones)")

    # Criterio 3: Zhang
    if row["zhang"] > 0.3:
        veredicto.append("✓ Zhang robusto (>0.3)")
        puntos += 2
    elif row["zhang"] > 0.1:
        veredicto.append("~ Zhang moderado")
        puntos += 1
    else:
        veredicto.append("✗ Zhang débil (≤0.1)")

    # Criterio 4: conviction
    if row["conviction"] > 1.5:
        veredicto.append("✓ Conviction alta (>1.5)")
        puntos += 1

    clasificacion = (
        "ROBUSTA" if puntos >= 6
        else "CANDIDATA" if puntos >= 3
        else "DESCARTAR"
    )

    if verbose:
        print(f"\nRegla: {row['regla']}")
        for v in veredicto:
            print(f"  {v}")
        print(f"  → CLASIFICACIÓN: {clasificacion} (puntos: {puntos}/8)")
    return clasificacion

# Evaluar top 5 reglas por lift
top_reglas = reglas_ext.sort_values("lift", ascending=False).head(5)
for _, row in top_reglas.iterrows():
    evaluar_regla(row)

Conclusión: Lo que Cambia con Este Marco

Antes (enfoque mecánico):
“Genero reglas, filtro por confidence > 0.5, reporto las top 10.”

Ahora (enfoque crítico):
“Examino lift, zhang y conviction. Verifico sensibilidad al umbral. Reporto solo las reglas que coinciden en múltiples métricas.”

Dimensión Antes Ahora
Métricas usadas 1–2 4–5
Dirección importa No Sí
Sensibilidad a parámetros Ignorada Explorada
Visualización Tabla Grafo + scatter
Veredicto Automático Sistemático

Tip

El analista que usa este marco genera menos reglas y más confiables.
Eso es mejor ciencia de datos.
