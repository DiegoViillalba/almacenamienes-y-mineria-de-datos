Objetivos de aprendizaje

¿Qué vamos a hacer hoy?

  • Diseñar y ejecutar un experimento de clasificación completo
  • Comparar múltiples clasificadores con la misma partición de datos
  • Identificar qué modelos requieren preprocesamiento específico
  • Interpretar métricas de forma comparativa, no aislada
  • Ajustar thresholds de decisión y observar su efecto
  • Reconocer errores comunes y retos reales del flujo de clasificación

Note

Nota Esta clase asume que ya se exploraron árboles de decisión (CART, ID3), Random Forest, métricas básicas (accuracy, precision, recall, F1) y la matriz de confusión. El énfasis está en la experimentación práctica, no en la teoría.

Evaluación: recordatorio operativo

Métricas: recordatorio compacto

\[\text{Accuracy} = \frac{TP+TN}{TP+TN+FP+FN}\]

\[\text{Precision} = \frac{TP}{TP+FP} \qquad \text{Recall} = \frac{TP}{TP+FN}\]

\[F_1 = \frac{2 \cdot \text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}\]

Note

Ya los conocen. Hoy los usamos como herramienta de comparación, no como tema de exposición.

Ajuste de thresholds

¿Por qué cambiar el threshold?

Por defecto, predict() usa threshold = 0.50.

Pero en diagnóstico médico: - Falso negativo = no detectar un tumor maligno → consecuencia grave - Falso positivo = alarma innecesaria → consecuencia menor

→ Queremos maximizar recall para la clase maligno
→ Eso significa bajar el threshold

Retos comunes y buenas prácticas

Errores frecuentes en experimentos de clasificación

  • ✗ Escalar antes del train/test split → data leakage
  • ✗ Evaluar solo con accuracy cuando hay desbalance
  • ✗ Comparar modelos con particiones distintas
  • ✗ No fijar random_state → resultados no reproducibles
  • ✗ Ajustar hiperparámetros mirando repetidamente el test set
  • ✗ Entrenar una red neuronal sin escalar las entradas
  • ✗ Comparar modelos con preprocesamientos inconsistentes
  • ✗ Modificar thresholds usando el test set como validación

Buenas prácticas

  • ✓ Usar pipelines para garantizar preprocesamiento consistente
  • ✓ Usar stratify al dividir datos
  • ✓ Evaluar siempre en datos no vistos durante entrenamiento
  • ✓ Reportar varias métricas, no solo accuracy
  • ✓ Analizar la matriz de confusión junto con las métricas
  • ✓ Considerar el costo relativo de FP y FN según el contexto
  • ✓ Documentar semillas, versiones y parámetros usados
  • ✓ Usar validación cruzada cuando el dataset es pequeño

La idea más importante

Entrenar un clasificador es fácil.
Diseñar bien el experimento es lo difícil.

  • La elección de métricas depende del contexto del problema
  • El threshold por defecto (0.5) no siempre es el mejor
  • Los pipelines no son opcionales: son obligatorios para reproducibilidad
  • El conjunto de prueba es sagrado: no lo uses para tomar decisiones de diseño
