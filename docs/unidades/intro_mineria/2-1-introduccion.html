3  Minería de Datos: Fundamentos Teóricos, Técnicas y Aplicaciones

Author

Facultad de Ciencias, UNAM

4 Introducción

La minería de datos constituye una disciplina formal dentro del proceso de descubrimiento de conocimiento en bases de datos (KDD). Su propósito no es simplemente consultar información almacenada, sino identificar patrones, regularidades, estructuras latentes y relaciones estadísticamente significativas dentro de grandes volúmenes de datos.

Desde un punto de vista metodológico, la minería de datos combina fundamentos de estadística inferencial, aprendizaje automático, optimización matemática y modelado computacional. Su desarrollo moderno se vincula con el crecimiento exponencial de datos digitales y con la necesidad de convertir información cruda en conocimiento accionable para la toma de decisiones.

5 Recuperación de datos versus minería de datos

Es fundamental distinguir entre recuperación de datos y minería de datos. La recuperación de datos implica ejecutar consultas donde el criterio de búsqueda está previamente definido. Por ejemplo, obtener todas las ventas realizadas en enero o listar pacientes diagnosticados con una enfermedad específica durante cierto año. En estos casos, la estructura de la pregunta está completamente determinada antes de interactuar con la base de datos.

En contraste, la minería de datos tiene como objetivo descubrir relaciones que no fueron explícitamente formuladas. Por ejemplo, determinar qué variables predicen la cancelación de citas médicas o identificar segmentos de clientes con mayor probabilidad de abandono. Aquí el sistema no solo responde a una condición predefinida, sino que infiere estructura a partir de los datos.

6 Correlación y causalidad

Uno de los principios epistemológicos más importantes en minería de datos es que la correlación no implica causalidad. Una asociación estadística entre dos variables no demuestra que una sea causa de la otra.

Por ejemplo, si se observa que estudiantes con mayor promedio académico tienden a tener mayores ingresos, esto únicamente indica una asociación. No puede concluirse que el promedio cause directamente el ingreso, pues podrían intervenir variables no observadas como contexto socioeconómico, red de contactos o tipo de institución.

Para establecer causalidad es necesario recurrir a diseños experimentales controlados. Una metodología clásica es la prueba A/B, donde los individuos son asignados aleatoriamente a un grupo control y a un grupo tratamiento, introduciendo una única diferencia y comparando resultados bajo una métrica previamente definida.

7 Clasificación (Aprendizaje Supervisado)

La clasificación es una técnica supervisada en la que existe una variable objetivo categórica. Formalmente, se busca aprender una función que asigne cada vector de características a una categoría discreta.

Matemáticamente, dado un conjunto de entrenamiento:

f: R^p → {1, …, K}

El modelo intenta minimizar el error de clasificación esperado.

Un ejemplo práctico consiste en predecir si un estudiante aprobará una materia utilizando variables como porcentaje de asistencia, número de tareas entregadas y horas de estudio semanal. En este caso, la variable objetivo es binaria (aprueba / no aprueba). Un modelo típico para este problema es la regresión logística.

Otro ejemplo corresponde a la clasificación de imágenes, donde a partir de características numéricas extraídas de píxeles se predice si una imagen corresponde a un gato o a un perro. En escenarios modernos, esta tarea se resuelve mediante redes neuronales profundas.

8 Regresión (Aprendizaje Supervisado)

La regresión se utiliza cuando la variable objetivo es numérica. El objetivo es modelar la relación entre variables independientes y una variable continua.

Un ejemplo clásico es la predicción del precio de una vivienda en función de sus características físicas y geográficas. El modelo lineal establece una relación aditiva entre variables explicativas y la variable objetivo, aunque también pueden emplearse modelos no lineales.

La evaluación de modelos de regresión se realiza mediante métricas como el error cuadrático medio (RMSE), el error absoluto medio (MAE) y el coeficiente de determinación R^2.

9 Clustering (Aprendizaje No Supervisado)

El clustering busca identificar grupos naturales dentro de los datos sin contar con etiquetas previas. El algoritmo más conocido es k-means, que minimiza la suma de distancias cuadráticas entre observaciones y los centroides de los grupos.

Un ejemplo aplicado consiste en segmentar usuarios de una plataforma digital según su comportamiento: número de sesiones por semana, tiempo promedio por sesión y gasto mensual. El resultado puede revelar perfiles como usuarios ocasionales, moderados o intensivos.

Es importante subrayar que los grupos no tienen significado intrínseco; su interpretación depende del contexto del problema.

10 Reglas de Asociación

Las reglas de asociación identifican patrones del tipo “si ocurre A, suele ocurrir B”. Se utilizan principalmente en análisis de canasta de mercado.

Las métricas fundamentales son soporte, confianza y lift. El soporte mide la frecuencia conjunta de A y B; la confianza estima la probabilidad condicional de B dado A; el lift compara dicha probabilidad con la frecuencia marginal de B.

Un ejemplo típico es identificar que quienes compran jarabe para la tos también compran pastillas para la garganta. Este tipo de conocimiento permite optimizar estrategias de venta cruzada.

11 Detección de Anomalías

La detección de anomalías identifica observaciones que se desvían significativamente del comportamiento normal. Es común en sistemas financieros para detectar fraude.

Por ejemplo, una transacción con monto inusualmente alto, realizada en horario atípico y desde una ubicación distante podría recibir una puntuación elevada de anomalía. Métodos como Isolation Forest o Local Outlier Factor son frecuentemente utilizados.

12 Proceso Integral

La minería de datos debe entenderse como un proceso estructurado que incluye definición del problema, preparación de datos, modelado, evaluación e implementación. Los algoritmos no sustituyen el conocimiento humano; requieren interpretación experta y validación rigurosa.

13 Consideraciones Éticas

El uso de modelos predictivos puede amplificar sesgos presentes en datos históricos. Es fundamental garantizar transparencia, explicabilidad y respeto a la privacidad. La implementación responsable es una obligación profesional.

14 Conclusiones

La minería de datos combina teoría estadística y técnicas computacionales para transformar datos en conocimiento. Sus principales técnicas incluyen clasificación, regresión, clustering, reglas de asociación y detección de anomalías. Sin embargo, su aplicación adecuada exige rigor metodológico, interpretación crítica y responsabilidad ética.
