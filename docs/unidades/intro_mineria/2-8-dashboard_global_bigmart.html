10  Dashboard global

11 Introducción

A lo largo de los capítulos anteriores se desarrolló una ruta progresiva para el análisis exploratorio del conjunto de datos Big Mart Sales. La lógica seguida no fue arbitraria: se construyó una descripción estadística de las ventas a partir de cinco dimensiones complementarias.

Primero se estudió la localización, para responder dónde se sitúan los valores típicos. Después se introdujo la variabilidad, para cuantificar la dispersión alrededor de esos centros. Más adelante se abordó la forma, con el fin de entender la asimetría, la curtosis y la estructura general de las distribuciones. Luego se examinó la heterogeneidad, es decir, las diferencias entre categorías, tipos de outlet y otros subgrupos. Finalmente, se incorporó la concentración, que permite analizar cómo se reparte el total de ventas entre las unidades que componen el sistema.

El propósito de este capítulo es integrar todas esas perspectivas en un dashboard global, útil tanto para la enseñanza del EDA como para la comunicación de hallazgos en un entorno de análisis aplicado. Este dashboard no sustituye el análisis detallado de los capítulos previos; más bien, actúa como una síntesis estructurada que permite recorrer, de manera compacta, la construcción progresiva de la medida.

12 Objetivos del capítulo

Este capítulo busca que el lector sea capaz de:

  1. integrar múltiples dimensiones del EDA en una misma narrativa analítica;
  2. construir indicadores resumidos y visualizaciones consistentes entre sí;
  3. diseñar un dashboard reproducible en Quarto compatible con un libro;
  4. interpretar de forma conjunta las dimensiones de localización, variabilidad, forma, heterogeneidad y concentración.

14 Arquitectura conceptual del dashboard

El dashboard global puede entenderse como una composición de cinco capas analíticas:

14.1 Capa 1. Localización

Resume el centro de la distribución mediante indicadores como media, mediana, moda y cuantiles.

14.2 Capa 2. Variabilidad

Introduce la amplitud y dispersión mediante rango, desviación estándar, IQR y coeficiente de variación.

14.3 Capa 3. Forma

Permite describir asimetría, curtosis y presencia de colas pesadas o valores atípicos.

14.4 Capa 4. Heterogeneidad

Compara la estructura de ventas entre subgrupos relevantes, como tipos de producto o tipos de outlet.

14.5 Capa 5. Concentración

Evalúa qué tan equilibrada o dominada está la estructura total de ventas.

En un dashboard pedagógico, estas cinco capas deben coexistir de forma coherente, de manera que el lector pueda pasar del nivel global al comparativo sin perder el hilo interpretativo.

16 Panel de indicadores globales

El primer componente del dashboard debe consistir en un bloque de KPIs generales que resuman el estado del conjunto de datos.

Ver código 
def safe_mode(series):
    m = series.mode(dropna=True)
    if len(m) == 0:
        return np.nan
    return m.iloc[0]

mean_sales = sales.mean()
median_sales = sales.median()
mode_sales = safe_mode(sales)
std_sales = sales.std(ddof=1)
iqr_sales = sales.quantile(0.75) - sales.quantile(0.25)
cv_sales = std_sales / mean_sales if mean_sales != 0 else np.nan
skew_sales = sales.skew()
kurt_sales = sales.kurt()

global_kpis = pd.DataFrame({
    "Indicador": [
        "Número de registros",
        "Ventas totales",
        "Media",
        "Mediana",
        "Moda",
        "Desviación estándar",
        "IQR",
        "Coeficiente de variación",
        "Asimetría",
        "Curtosis"
    ],
    "Valor": [
        len(df),
        round(sales.sum(), 2),
        round(mean_sales, 2),
        round(median_sales, 2),
        round(mode_sales, 2) if pd.notnull(mode_sales) else np.nan,
        round(std_sales, 2),
        round(iqr_sales, 2),
        round(cv_sales, 4),
        round(skew_sales, 4),
        round(kurt_sales, 4)
    ]
})

17 Panel de localización y forma

Una forma natural de combinar localización y forma es mediante un histograma enriquecido con líneas de referencia para la media y la mediana.

Ver código 
fig = px.histogram(
    df,
    x="Item_Outlet_Sales",
    nbins=50,
    marginal="box",
    title="Distribución global de Item_Outlet_Sales"
)

fig.add_vline(x=mean_sales, line_dash="dash", annotation_text="Media")
fig.add_vline(x=median_sales, line_dash="dot", annotation_text="Mediana")

fig.update_layout(
    xaxis_title="Ventas",
    yaxis_title="Frecuencia"
)
fig.show()

17.1 Interpretación

Este panel sintetiza dos preguntas esenciales:

  • ¿Dónde se localizan los valores típicos?
  • ¿La distribución es aproximadamente simétrica o presenta sesgo?

Cuando la media y la mediana difieren de manera visible, y cuando la cola derecha es más larga, suele haber evidencia de asimetría positiva. En un entorno como Big Mart Sales, esto es compatible con la existencia de observaciones con ventas muy elevadas respecto del grueso de la distribución.

20 Panel de concentración

Para el dashboard global conviene incorporar una visualización compacta de concentración. El análisis por tipo de producto suele ser especialmente ilustrativo.

Ver código 
sales_by_item_type = (
    df.groupby("Item_Type")["Item_Outlet_Sales"]
      .sum()
      .sort_values(ascending=False)
      .reset_index()
)

sales_by_item_type["share"] = (
    sales_by_item_type["Item_Outlet_Sales"] /
    sales_by_item_type["Item_Outlet_Sales"].sum()
)
sales_by_item_type["cum_share"] = sales_by_item_type["share"].cumsum()
sales_by_item_type["share_pct"] = 100 * sales_by_item_type["share"]
sales_by_item_type["cum_share_pct"] = 100 * sales_by_item_type["cum_share"]

def categories_to_reach_threshold(table, threshold=0.8):
    idx = np.argmax(table["cum_share"].to_numpy() >= threshold)
    return idx + 1

21 Ensamblaje del dashboard global

En un libro de Quarto, el dashboard puede presentarse como una secuencia ordenada de paneles. Una estructura práctica es la siguiente:

  1. Bloque inicial de KPIs: resumen numérico de alto nivel.
  2. Distribución global: histograma y boxplot para localización, forma y variabilidad.
  3. Comparación entre grupos: heterogeneidad por outlet y por producto.
  4. Concentración: Pareto y tabla de acumulación.
  5. Texto interpretativo: síntesis narrativa de hallazgos.

21.1 Ejemplo de lectura integrada

Desde una perspectiva estadística, el dashboard global permite articular una lectura de este tipo:

  • La distribución global de ventas presenta un centro identificable, pero también una dispersión importante.
  • La separación entre media y mediana, junto con la cola derecha, sugiere asimetría positiva.
  • Los boxplots muestran variabilidad considerable y presencia de valores atípicos.
  • Las comparaciones por tipo de outlet y tipo de producto revelan heterogeneidad entre subgrupos.
  • El análisis de concentración muestra que una fracción limitada de categorías contribuye de forma desproporcionada al total.

Esta lectura resume precisamente la lógica del EDA progresivo: la medida no se agota en un único número, sino que se construye mediante varias capas complementarias.

23 Conclusiones

El dashboard global no debe entenderse como una simple colección de gráficas, sino como una síntesis argumentada del EDA. Su valor reside en que integra distintos niveles de descripción en una sola narrativa:

  • el centro de la distribución,
  • su amplitud,
  • su geometría,
  • sus diferencias entre subgrupos,
  • y su estructura de reparto.

En el caso de Big Mart Sales, esta integración permite comunicar de forma compacta cómo se comportan las ventas desde varios ángulos complementarios. Para fines docentes, el dashboard muestra que la estadística descriptiva es una construcción progresiva de la medida; para fines aplicados, ofrece un instrumento claro de exploración y comunicación.

24 Sugerencias de extensión

  1. Añadir filtros interactivos por Outlet_Type, Outlet_Size o Item_Fat_Content.
  2. Integrar una sección de correlación entre variables numéricas.
  3. Incorporar mapas si se dispone de georreferenciación de outlets.
  4. Conectar el dashboard con la fase posterior de modelado predictivo.
