1  Almacenes de Datos

1.1 Introducción y conceptos básicos

Un Almacén de Datos (Data Warehouse) es una colección de datos orientada a temas, integrada, variante en el tiempo y no volátil, diseñada para apoyar los procesos de toma de decisiones en una organización.

A diferencia de las bases de datos transaccionales (OLTP), un almacén de datos se enfoca en el análisis histórico y multidimensional.

1.2 Características de un Almacén de Datos

Siguiendo la definición clásica de Bill Inmon, las cuatro características fundamentales son:

  1. Orientado a temas: Los datos se organizan alrededor de los sujetos principales de la empresa (ej. clientes, productos, ventas) en lugar de procesos operativos.
  2. Integrado: Se construye mediante la integración de múltiples fuentes heterogéneas (bases de datos relacionales, archivos de texto, etc.), asegurando consistencia en nombres, unidades y estructuras.
  3. Variante en el tiempo: Los datos se guardan como una serie de snapshots históricos. Cada registro incluye un elemento de tiempo explícito.
  4. No volátil: Una vez que los datos entran al almacén, no se modifican ni se eliminan; el acceso es principalmente de lectura.

1.3 Arquitectura de un Almacén de Datos

La arquitectura típica consta de varias capas que facilitan el flujo de la información:

  • Sistemas Fuente: Bases de datos operacionales y archivos externos.
  • Área de Pruebas (Staging Area): Donde se realiza la limpieza y transformación.
  • Almacén de Datos Corporativo: El repositorio central de datos integrados.
  • Data Marts: Subconjuntos de datos específicos para departamentos (ej. Data Mart de Finanzas).
  • Herramientas de Análisis: OLAP, minería de datos y reporting.

1.4 El proceso de ETL (Extracción, Transformación y Carga)

Este es el proceso crítico para alimentar el almacén:

  1. Extracción: Obtención de datos desde las fuentes originales.
  2. Transformación: Limpieza, filtrado, validación y conversión de formatos para asegurar la calidad del dato.
  3. Carga: Inserción de los datos transformados en la estructura final del almacén.

1.5 Modelado Multidimensional

A diferencia del modelo relacional normalizado, aquí se utilizan estructuras optimizadas para consultas complejas:

  • Tabla de Hechos (Fact Table): Contiene las métricas cuantitativas o medidas del negocio.
  • Tablas de Dimensiones: Contienen los atributos descriptivos que contextualizan los hechos (Tiempo, Geografía, Producto).

1.5.1 Esquemas comunes:

  • Esquema de Estrella: Una tabla de hechos central conectada directamente a tablas de dimensiones.
  • Esquema de Copo de Nieve (Snowflake): Una variante donde las tablas de dimensiones están normalizadas.
